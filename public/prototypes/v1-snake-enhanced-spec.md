# 贪吃蛇增强版 - 产品原型设计文档

> 版本：v1.0 | 日期：2026-03-24 | 状态：原型

---

## 概述

本项目为经典贪吃蛇游戏的增强版，保留核心玩法，叠加以下功能模块：

- 最高分记录（localStorage 持久化）
- 成就徽章系统
- 皮肤切换系统
- 8-bit 音效 + 手机震动
- 局域网双人对战（Phase 5）

**核心原则：** 不改变原版贪吃蛇的玩法本质，所有增强均为可选项，不影响原用户体验。

## 功能模块规格

### Phase 1: 最高分记录

**用户故事：** 玩家每次通关后能看见自己的历史最高分，激励重复挑战。

**数据模型：**
```ts
// localStorage key: 'snake_highscore'
type HighScore = {
  score: number;       // 最高分
  updatedAt: number;   // timestamp
}
```

**UI 位置：**
- 游戏主画面：得分卡右侧新增"最高分：XXX"显示
- 新纪录诞生时：得分卡闪烁绿色 + "NEW!" 标签

**交互细节：**
- 首次打开：最高分为 0
- 游戏结束时：比较 score 和 highScore，大的话更新并同步 localStorage
- 无网络依赖，纯客户端

---

### Phase 2: 成就系统

**用户故事：** 给玩家设置可追逐的里程碑目标，增加长期留存动力。

**成就列表：**

| ID | 名称 | 描述 | 解锁条件 |
|----|------|------|----------|
| `first_bite` | 初尝禁果 | 第一次吃到食物 | score 从 0→1 |
| `gourmet_10` | 小有名气 | 单局得 10 分 | score ≥ 10 |
| `gourmet_30` | 蛇吞百味 | 单局得 30 分 | score ≥ 30 |
| `gourmet_50` | 贪食之王 | 单局得 50 分 | score ≥ 50 |
| `half_board` | 半壁江山 | 蛇身占据棋盘一半 | snake.length ≥ GRID_SIZE²/2 |
| `perfect_fill` | 完美收官 | 蛇身填满整个棋盘 | snake.length === GRID_SIZE² |
| `comeback_kid` | 绝地反击 | 落后 5 分后翻盘获胜 | 落后→反超 |
| `no_miss` | 百发百中 | 连续吃 10 个食物无失误 | 连续 10 次 eat |
| `speedster` | 极速反应 | 在 1 秒内连续吃 2 个食物 | 两次 eat 间隔 < 1000ms |

**存储：**
```ts
// localStorage key: 'snake_achievements'
type AchievementStore = {
  [id: string]: {
    unlockedAt: number; // timestamp，undefined 表示未解锁
  }
}
```

**UI 位置：**
- 游戏结束画面：下方新增"成就"区域（3x3 网格图标）
- 已解锁：彩色图标 + 名称 + 解锁时间
- 未解锁：灰色锁定图标 + 问号
- 新解锁：金色边框 + 弹窗提示

---

### Phase 3: 皮肤系统

**用户故事：** 满足个性化审美需求，提升游戏的视觉新鲜感。

**皮肤列表：**

| ID | 名称 | 蛇头颜色 | 蛇身颜色 | 食物颜色 | 棋盘背景 |
|----|------|----------|----------|----------|----------|
| `default` | 经典绿 | `#22c55e` | `#15803d` | `#f97316` | `#020617` |
| `neon` | 霓虹夜 | `#06b6d4` | `#6366f1` | `#f472b6` | `#0a0a1a` |
| `pixel` | 像素复古 | `#dc2626` | `#7c2d12` | `#fbbf24` | `#1c1917` |
| `candy` | 糖果风 | `#f9a8d4` | `#db2777` | `#a3e635` | `#2e1065` |
| `mario` | 超级马里奥 | `#ef4444` | `#1d4ed8` | `#fbbf24` | `#fef3c7` |

**存储：**
```ts
// localStorage key: 'snake_skin'
type SkinStore = {
  selectedSkin: SkinId;
  unlockedSkins: SkinId[]; // 初始只有 'default' 解锁
}
```

**UI 位置：**
- 游戏结束画面：右侧新增"皮肤"选择栏（横向滚动）
- 每个皮肤：圆形预览 + 名称标签
- 未解锁皮肤：锁定状态，点击弹出"达成所有成就解锁"

---

### Phase 4: 音效与震动

**技术方案：** Web Audio API（无外部音频文件，8-bit 程序化生成）

| 事件 | 音效描述 | 频率/时长 |
|------|----------|-----------|
| 吃食物 | 上升音调 | 440Hz → 880Hz，100ms |
| 死亡 | 下沉音调 | 400Hz → 100Hz，300ms |
| 新纪录 | 胜利旋律 | C5-E5-G5-C6，500ms |
| 成就解锁 | 叮咚提示 | 660Hz + 880Hz 各 80ms |
| 按键 | 轻触反馈 | 200Hz，20ms |

**震动：** `navigator.vibrate(30)` 在吃食物时触发（仅移动端）

---

### Phase 5: 局域网双人对战（Phase 5）

**用户故事：** 与朋友同屏对战，增加社交乐趣。

**架构：**
- WebSocket 点对点连接
- 其中一人创建房间（生成 4 位数字码），另一人输入房间码加入
- 各自跑相同游戏逻辑，仅同步：对方蛇头位置、方向、对方得分
- 本地渲染自己的蛇 + 对方的蛇头（半透明）

**状态同步协议：**
```ts
type SyncMessage = {
  type: 'join' | 'move' | 'score_update' | 'game_over';
  playerId: string;
  direction?: Direction;
  score?: number;
  snakeHead?: Cell;
}
```

**冲突处理：** 吃到同一食物时，距食物更近的优先，远的收到 `food_eaten` 通知后重新生成。

## 技术约束

- **不改变** `advanceGame.ts` 纯函数层
- **不改变** `gridHelpers.ts` 纯函数辅助层
- 所有增强功能在 `useSnakeGame.ts`（副作用）和 `SnakeGame.tsx`（UI）层实现
- 皮肤系统通过 CSS 变量注入，不修改 JSX 结构
- 音效通过 Web Audio API 程序化生成，不引入音频文件依赖
- 双人对战阶段需额外 npm 包（`ws`），独立部署
